# دليل البدء السريع — Dasm Inspection

---

## 1. محتويات المستودع الحالية

المستودع يحتوي حالياً على:

- **`md-files/`** — حزمة التوثيق والتحليل (هذا المجلد).
- **`Dasm inspection.make`** — تصدير Figma Make (إن وُجد).

لا يوجد كود تنفيذي بعد؛ هذه مرحلة توثيق وتحليل فقط.

---

## 2. كيف تقرأ التوثيق؟

### ترتيب مقترح

1. **`DELIVERY_SUMMARY.md`** — لفهم ما تم إنجازه والخطوات التالية.
2. **`README.md`** — لفهم المنتج والشاشات.
3. **`PROJECT_ANALYSIS.md`** — للتحليل المعماري والشاشات والمكونات.
4. **`CURSOR_INSTRUCTIONS.md`** — للتعليمات التقنية عند البدء بالتنفيذ.
5. **`TODO.md`** — لقائمة المهام والمراحل.
6. **`COLLABORATION_GUIDE.md`** — لأسلوب التعاون مع Cursor و Claude.
7. **`INDEX.md`** — للوصول السريع لأي ملف.

### متى ترجع لكل ملف؟

- **قبل كتابة أي كود:** `CURSOR_INSTRUCTIONS.md` و `PROJECT_ANALYSIS.md`
- **لفهم ما الذي يجب تنفيذه:** `TODO.md`
- **عند الحاجة لإرشاد Cursor:** راجع `INDEX.md` وقسم "كيف أستخدم هذا التوثيق مع Cursor؟"

## 3. المتطلبات المسبقة (للمرحلة القادمة)

عند بدء التنفيذ ستكون هناك حاجة لـ:

- Node.js 18+
- npm أو pnpm
- حساب Vercel (للنشر)
- حساب Supabase (عند ربط Backend)

حالياً لا يُطلب تنفيذ أي أمر؛ التوثيق فقط.
